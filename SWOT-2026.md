# SWOT / SÜOT Analysis (Apr 2026)

## Product: Mindshift Academy (8-12, single-device MVP)

## Strengths (S)

- Age-banded UX (8-9 / 10-11 / 12) lowers cognitive overload and improves completion likelihood.
- Safe gamification strategy (missions, progress, badges) avoids toxic competitive mechanics.
- Offline-first architecture (single HTML + localStorage) removes infra cost and launch friction.
- Parent layer (weekly summary + checklist) creates scaffolding and accountability around learning.
- Focus-session mechanic supports attention management for children (short cycles).
- Accessibility baseline improved (focus-visible, larger targets, reduced motion support).

## Weaknesses (W)

- No cloud sync: progress is isolated per device/browser.
- No formal automated E2E test suite yet (manual QA exists, but not CI-enforced).
- AI interaction still route-based; no secure backend prompt orchestration/policy guardrail yet.
- Content quality depends on periodic pedagogical review; no expert review pipeline in-app.
- LocalStorage model can be cleared by browser user action.

## Opportunities (O)

- Add free-tier backend (Supabase) for optional multi-device sync and parent dashboards.
- Add privacy-preserving analytics (event counters without PII) to improve retention decisions.
- Expand content for localized tracks (AZ primary, optional RU/EN overlays).
- Build partner package for schools/learning centers with teacher mode.
- Add adaptive pathing based on mastery data (lightweight rule engine).

## Threats (T)

- AI output quality/safety variability may reduce trust if not moderated and contextualized.
- Policy/privacy compliance risk for children under 13 if backend data collection expands.
- Fast-moving competitors with larger content teams may outpace feature breadth.
- Platform policy changes (GitHub Pages, API policy, browser storage behavior) can affect ops.
